- `dct:source` mandatory + repeatable (3 values per instance: Dataset FDO + FAIR² Package + FAIR² resource URL — per SPS v1.3 §"Source URLs (Types 3-6)").
- `cito:citesAsAuthority` **absent** per demonstration posture — two-hop attribution via the method FDO; the method instances carry the published authority DOIs.
- `mac:isObservationOf` → Type 1 anchor (Alpha / Epsilon / Eta) — mandatory.
- `mac:hasPredictionMethod` → Type 9 method (ESM2 / AlphaFold2) — mandatory.
- Self-materialization 4-statement block (st5/5b/5c/5d): byte-identical to Type 1 + method instances on all 18.
- `dct:isPartOf` → draft StayAhead Project FDO on all 18 (per Post_demonstration_cleanup.md).
- Each instance's pubinfo carries `nt:wasCreatedFromTemplate` pointing at the matching Stage 2 Type 3 / 4 / 5 template Trusty URI.

## Scale-up readiness

The CSV-driven path (`generate_predictions_from_csv.py`) is the entry
point for the Tranche 2 expansion to ~3705 mutations:
- New variants only require adding entries to `VARIANT_ANCHORS` (after their Type 1 anchor FDOs are published).
- Methods unchanged unless a new predictor is introduced.
- The driver iterates the CSV and emits one TriG per (variant × method × metric).
- At 3 metrics × the published methods, 3705 mutations × 2 methods = ~22 230 prediction instances per metric set, or ~66 690 total — same code path, no per-instance config files needed.

## Note on `dct:source` Dataset FDO references

Each prediction instance references a draft Dataset FDO (ESM dataset
`RA9YxiAB…` or AlphaFold dataset `RAv6JJe…`) via `dct:source`. Per
`Post_demonstration_cleanup.md`, both Datasets will be promoted from
`npx:DraftNanopub` to non-draft in the coordinated revision pass. At
that point the references can be updated to the supersession heads via
instance supersession (or left as-is — `prov:wasDerivedFrom` and
`npx:supersedes` make the chain navigable either way).
